发表于 2018-07-24 | 分类于 其他

环境配置

在开始之前，我们需要检测电脑中是否安装以下环境，如果没有安装可以到下面的网站中自行安装：

  • Node.js
  • Git

如果是Linux系统的话，这里介绍一种更简单快捷的方式来安装环境，那就是Homebrew。点击链接按照提示安装Homebrew。然后通过Homebrew进行安装我们所需要的环境配置。

建站&配置

安装完环境之后，我们就要开始正式的建立网站了，执行以下命令，将网站存放在本地文件夹中：

完成之后，会发现在选中的文件夹中生成了一系列文件，暂时有三个文件对我们游泳。

_config.yml

网站的配置信息在这里，我们可以进行参数的设置，来个性化网站。比如可以设置网站标题，副标题，名字等等：

source

网站的内容在这里，打开source文件夹，会发现有一个 _posts已经存在，后续我们写的文章都会在这里。还可以在这里配置分类和标签。

themes

网站的主题在这里，我们可以先把主题下载下来，存放在这里，然后通过配置参数来修改网站的主题。网站的主题也可以进行一些个性化操作。进入到具体的主题文件夹后打开 _config.yml文件进行相关配置。

这样一系列操作以后，我们的网站已经初步完成了，但是这个时候还是在本地，接下来很重要的一步就是发布到GitHub上，让我们在外网可以访问。

GitHub Pages

原本我们需要自己购买云服务，才能将网站部署上去，但是现在可以通过GitHub Pages，将我们的网站部署到GitHub上去，GitHub会给我们生产一个唯一的域名来访问网站。这是GitHub Pages的介绍和使用：

  • GitHub Pages

按照教程中的步骤很容易将你的GitHub个人网页配置好，但这时只是一个空的网页，接下来我们就要将我们本地的网站和这个网页链接到一起。
